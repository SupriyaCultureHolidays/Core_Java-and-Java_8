Two Pointer

Two Pointer হলো DSA-র একটা খুবই গুরুত্বপূর্ণ Pattern — মূলত Array এবং String-এর Problem Solve করতে ব্যবহার হয়। সহজ কথায়: একটা Array/String-এর উপর কাজ করার জন্য দুটো Index (Pointer) ব্যবহার করা হয়, যাতে বারবার একই Element Check না করে কম সময়ে Solution বের করা যায়।

কেন Two Pointer ব্যবহার করি

ধরো একটা Array আছে [1, 2, 3, 4, 5]। সব Pair Check করতে চাইলে (1-এর সাথে সবাই, 2-এর সাথে সবাই...) Nested Loop লাগবে। কিন্তু Two Pointer ব্যবহার করলে অনেক ক্ষেত্রেই একই কাজ কম সময়ে হয়ে যায়।

ApproachTime Complexity
Nested Loop (Brute Force)O(n²)
Two PointerO(n)

Two Pointer-এর প্রধান ৩টা Pattern

১) Slow & Fast Pointer — একই দিকে এগোয়

দুটো Pointer একই Direction-এ (বাম থেকে ডানে) এগোয়। i (Slow) এখন পর্যন্ত পাওয়া Unique জায়গা ধরে রাখে, j (Fast) পুরো Array ঘুরে দেখে।

Remove Duplicate — Dry Run
Array: [1, 1, 2, 2, 3]

i = 0, j = 1  →  nums[i]=1, nums[j]=1  →  Same, কিছু করার নেই
j++           →  nums[i]=1, nums[j]=2  →  Different  →  i++, nums[i] = nums[j]
              →  Array হয়ে যায়: [1, 2, 2, 2, 3]
              →  প্রথম দুটো ঘর এখন Unique: [1, 2]

কোথায় ব্যবহার হয়: Remove Duplicates, Remove Element, Move Zeroes, Array Compression

২) Left & Right Pointer — দুই প্রান্ত থেকে

একটা Pointer শুরুতে (left), আরেকটা শেষে (right) থাকে — দুজন একে অপরের দিকে এগোয়।

Reverse Array
[1, 2, 3, 4, 5]

1 ↔ 5  →  [5, 2, 3, 4, 1]
2 ↔ 4  →  [5, 4, 3, 2, 1]

কোথায় ব্যবহার হয়: Reverse Array, Palindrome Check, Two Sum (Sorted Array), Container With Most Water

৩) Sliding Window — Two Pointer-এর Advanced রূপ

দুটো Pointer মিলে একটা "Window" বানায় — দরকারমতো বড় বা ছোট হয়।

Window বড়/ছোট হওয়া
[1, 2, 3, 4, 5]

L R
[1, 2]        ← Window ছোট

L     R
[1, 2, 3]     ← Window বড় হলো

   L  R
   [2, 3]     ← দরকার হলে আবার ছোট হলো

কোথায় ব্যবহার হয়: প্রশ্নে "Longest / Smallest / Maximum / Minimum Subarray বা Substring" থাকলে — যেমন "Longest Substring Without Repeating Characters"

কীভাবে বুঝবে কোন Pattern লাগবে

  1. Array কি Sorted? → Two Pointer Try করো
  2. In-place Change করতে হবে (নতুন Array বানানো যাবে না)? → Slow-Fast Pointer
  3. Pair খুঁজতে হবে (Sorted Array-তে)? → Left-Right Pointer
  4. Continuous অংশ (Subarray/Substring) খুঁজতে হবে? → Sliding Window
Problem TypePattern
Duplicate RemoveSlow-Fast Pointer
Element RemoveSlow-Fast Pointer
Zero MoveSlow-Fast Pointer
ReverseLeft-Right Pointer
PalindromeLeft-Right Pointer
Pair Sum (Sorted Array)Left-Right Pointer
Longest SubstringSliding Window
Interview Rule: Array/String দেখলেই আগে ভাবো — Sorted? → Two Pointer। In-place? → Slow-Fast। Pair? → Left-Right। Subarray/Substring? → Sliding Window।

এই ৬টা করলে Two Pointer-এর Base অনেক Strong হয়ে যাবে — এগুলো Array আর String দুটো Section-এর Roadmap-এও আছে, এখানে শুধু Two Pointer-এর Lens দিয়ে একসাথে দেখানো হলো।

Array Traversal — Loop দিয়ে ঘোরা

DSA-তে সবচেয়ে বেশি ব্যবহৃত Operation।

সাধারণ for loop

    System.out.println(arr[i]);

for (int value : arr) {
    System.out.println(value);

পার্থক্য: সাধারণ Loop-এ Index (i) পাওয়া যায় — তাই Value বদলাতে বা নির্দিষ্ট Index লাগলে এটা লাগবে। Enhanced Loop-এ শুধু Value পাওয়া যায়, Index পাওয়া যায় না।

Scanner দিয়ে Array Input নেওয়া

n সংখ্যক input নিয়ে Array ভরা
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int[] arr = new int[n];

        for (int i = 0; i < n; i++) {
            arr[i] = sc.nextInt();
        }

        for (int i = 0; i < n; i++) {
            System.out.print(arr[i] + " ");
        }
    }
}

Reverse করা — Two Pointer

Two Pointer — visual

Two Pointer দিয়ে Reverse
int left = 0, right = arr.length - 1;

while (left < right) {
    int temp   = arr[left];
    arr[left]  = arr[right];
    arr[right] = temp;

    left++;
    right--;
}

String

Array skip করে সরাসরি String-এ ঝাঁপ দিও না। DSA-র অনেক String Problem আসলে Array-এর মতোই Two Pointer আর Sliding Window Pattern ব্যবহার করে — Array-এর Basic শক্ত না থাকলে এখানে সমস্যা হবে।

Java-তে String একটি Class, Primitive নয় — এবং এটি Immutable (একবার তৈরি হলে বদলানো যায় না)। ভেতরে এটি আসলে একটি char[]-এর উপর তৈরি, কিন্তু char[]-এর মতো সরাসরি index দিয়ে বদলানো যায় না।

char[] — Mutable
char[] arr = {'h','i'};
arr[0] = 'H';
// সরাসরি বদলানো যায়
String — Immutable
String s = "hi";
// s.charAt(0) = 'H';  ❌ এভাবে বদলানো যায় না
s = "Hi";   // নতুন String তৈরি হয়

Immutability উদাহরণ

03

Recursion

আগে Array ও String-এর Basic Problem শেষ করে এসো। Recursion-এর অনেক Problem আসলে Array/String-এর উপরেই হয় — এবং Recursion ঠিকভাবে শিখলে পরে Tree, Backtracking ও Dynamic Programming অনেক সহজ লাগবে।

Recursion মানে — একটি Function নিজেই নিজেকে Call করে। প্রতিটা Recursive Function-এ দুটো অংশ থাকতেই হয়: একটা Base Case (যেখানে থেমে যায়) এবং একটা Recursive Case (যেখানে নিজেকে আবার Call করে, কিন্তু ছোট Input নিয়ে)।

Factorial — Base Case + Recursive Case
int factorial(int n) {
    if (n == 0) return 1;        // Base Case — এখানে থেমে যায়
    return n * factorial(n - 1); // Recursive Case — ছোট Input নিয়ে নিজেকে Call করে
}

Call Stack — Recursion Memory-তে কীভাবে কাজ করে

প্রতিটা Recursive Call-এর জন্য একটা নতুন Stack Frame তৈরি হয়, ঠিক সাধারণ Method Call-এর মতোই। যতক্ষণ Base Case না পাওয়া যায়, ততক্ষণ নতুন Frame Push হতেই থাকে — তারপর Base Case থেকে একে একে Frame Pop হয়ে Result উপরে ফেরত যায়।

CALL STACK — factorial(3)
▲ factorial(0) ← সবার শেষে Push, সবার আগে Pop (Base Case)
return=1
factorial(1)
n=1// অপেক্ষায় — 1 * factorial(0)
factorial(2)
n=2// অপেক্ষায় — 2 * factorial(1)
▼ factorial(3) ← সবার আগে Push, সবার শেষে Pop
n=3// অপেক্ষায় — 3 * factorial(2)
Unwind — Base Case থেকে ফেরত আসা
factorial(3)
= 3 * factorial(2)
= 3 * (2 * factorial(1))
= 3 * (2 * (1 * factorial(0)))
= 3 * (2 * (1 * 1))
= 6

Stack Overflow

Base Case না থাকলে বা ভুল Base Case লিখলে Function নিজেকে থামা ছাড়াই Call করতে থাকে — প্রতিটা Call-এর জন্য নতুন Frame Push হয়, একসময় Stack ভরে গিয়ে Error আসে।

Base Case ছাড়া — ভুল Recursion
int badRecursion(int n) {
    return badRecursion(n - 1);   // Base Case নেই — কখনো থামবে না
}

// Exception in thread "main" java.lang.StackOverflowError
নিয়ম: Base Case সবসময় সবার আগে লেখো, এবং নিশ্চিত করো প্রতিটা Recursive Call যেন Base Case-এর দিকে এগিয়ে যায় (যেমন n ছোট হতে হতে 0-তে পৌঁছাবে)।

Dry Run — সবচেয়ে গুরুত্বপূর্ণ Skill

Dry Run মানে কাগজে-কলমে বা মনে মনে প্রতিটা Call ধাপে ধাপে Trace করা। Recursion বোঝার এটাই আসল উপায় — Code দেখে না বুঝলে, ছোট Input নিয়ে হাতে করে দেখো।

  1. ছোট একটা Input নাও (যেমন n = 3 বা 4)
  2. প্রতিটা Call আলাদা লাইনে লিখে Stack-এর মতো সাজাও
  3. Base Case না পাওয়া পর্যন্ত Call চালিয়ে যাও
  4. Base Case থেকে Return শুরু করে, Unwind করে উপরে ওঠো
  5. প্রতি ধাপে Return Value মিলিয়ে দেখো — এখান থেকেই Pattern চোখে পড়বে

04

Linked List

Recursion না শিখে এখানে এসো না। Linked List-এর অনেক Solution (Reverse, Merge, Recursive Traversal) Recursion-এর উপর নির্ভর করে। তবে প্রথমে Iterative Solution শিখবে, তারপর Recursive।

Linked List হলো একগুচ্ছ Node-এর Chain — প্রতিটা Node-এ থাকে Data এবং পরের Node-এর ঠিকানা (next)। Array-এর মতো Continuous Memory লাগে না — প্রতিটা Node Heap-এর যেকোনো জায়গায় থাকতে পারে, শুধু next দিয়ে একটার সাথে আরেকটা জোড়া লাগানো থাকে।

Node — Linked List-এর একক
class Node {
    int data;
    Node next;

    Node(int data) {
        this.data = data;
        this.next = null;
    }
}
তিনটে Node জোড়া লাগানো
Node head = new Node(10);
head.next = new Node(20);
head.next.next = new Node(30);

// Traverse এবং Print
Node temp = head;
while (temp != null) {
    System.out.print(temp.data + " → ");
    temp = temp.next;
}
System.out.println("null");   // 10 → 20 → 30 → null
HEAP — Node গুলো ছড়িয়ে ছিটিয়ে, next দিয়ে জোড়া লাগানো
Node 1 (Head)
data=10
next=ref →
Node 2
data=20
next=ref →
Node 3 (Tail)
data=30
next=null
Head মানে প্রথম Node-এর Reference — এটা হারিয়ে গেলে পুরো Linked List হারিয়ে যায়। Tail মানে শেষ Node, যার next সবসময় null।

গুরুত্বপূর্ণ Pattern ১ — Slow & Fast Pointer

Middle Node খুঁজতে বা Loop Detect করতে সবচেয়ে ব্যবহৃত Pattern। slow এক ঘর করে এগোয়, fast দুই ঘর করে — fast শেষে পৌঁছালে slow ঠিক Middle-এ থাকে।

Middle Node খোঁজা
Node slow = head, fast = head;

while (fast != null && fast.next != null) {
    slow = slow.next;         // ১ ঘর
    fast = fast.next.next;    // ২ ঘর
}
// fast শেষে পৌঁছালে slow মাঝখানে
Floyd's Cycle Detection: একই Slow-Fast Pointer দিয়ে Loop আছে কিনা বোঝা যায় — Loop থাকলে fast কোনো একসময় ঘুরে এসে slow-কে ধরে ফেলবে (slow == fast)।

গুরুত্বপূর্ণ Pattern ২ — Reverse Linked List (Iterative)

সবচেয়ে বেশি জিজ্ঞেস করা Interview Question। তিনটে Pointer লাগে — prev, curr, next।

prev / curr / next — তিনটে Pointer
Node prev = null;
Node curr = head;

while (curr != null) {
    Node next = curr.next;   // পরের Node হারিয়ে যাওয়ার আগে ধরে রাখো
    curr.next = prev;        // Link উল্টে দাও
    prev = curr;              // prev এগিয়ে যাক
    curr = next;              // curr এগিয়ে যাক
}

head = prev;   // নতুন Head এখন শেষের Node

Stack

Stack আসলে একটা Pattern-Based Data Structure। এটা ভালোভাবে শিখলে পরে Queue, Trees (DFS), Graph (DFS), Recursion এবং Expression-এর Problem অনেক সহজ হয়ে যাবে।

Stack একটা Data Structure যেটা LIFO (Last In First Out) নিয়মে চলে — সবচেয়ে শেষে যেটা রাখা হয়েছে, সেটাই সবার আগে বের হয়। প্লেটের Stack-এর মতো — উপরের প্লেটটাই সবার আগে সরানো হয়।
