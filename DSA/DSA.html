n সংখ্যক input নিয়ে Array ভরা
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int[] arr = new int[n];

        for (int i = 0; i < n; i++) {
            arr[i] = sc.nextInt();
        }

        for (int i = 0; i < n; i++) {
            System.out.print(arr[i] + " ");
        }
    }
}

Two Pointer দিয়ে Reverse
int left = 0, right = arr.length - 1;

while (left < right) {
    int temp   = arr[left];
    arr[left]  = arr[right];
    arr[right] = temp;

    left++;
    right--;
}

02

String

Array skip করে সরাসরি String-এ ঝাঁপ দিও না। DSA-র অনেক String Problem আসলে Array-এর মতোই Two Pointer আর Sliding Window Pattern ব্যবহার করে — Array-এর Basic শক্ত না থাকলে এখানে সমস্যা হবে।

Java-তে String একটি Class, Primitive নয় — এবং এটি Immutable (একবার তৈরি হলে বদলানো যায় না)। ভেতরে এটি আসলে একটি char[]-এর উপর তৈরি, কিন্তু char[]-এর মতো সরাসরি index দিয়ে বদলানো যায় না।

char[] — Mutable
char[] arr = {'h','i'};
arr[0] = 'H';
// সরাসরি বদলানো যায়
String — Immutable
String s = "hi";
// s.charAt(0) = 'H';  ❌ এভাবে বদলানো যায় না
s = "Hi";   // নতুন String তৈরি হয়

গুরুত্বপূর্ণ String Method — একনজরে

Methodকী করেউদাহরণ
length()মোট Character সংখ্যা"hello".length() → 5
charAt(i)i-তম Index-এর Character"hello".charAt(1) → 'e'
substring(a,b)a থেকে b-1 পর্যন্ত অংশ"hello".substring(1,4) → "ell"
equals() vs ==equals() Value তুলনা করে, == Reference তুলনা করেs1.equals(s2)
compareTo()Lexicographically তুলনা — Negative/0/Positive ফেরত দেয়"a".compareTo("b") → Negative
toUpperCase() / toLowerCase()সব Character বড়/ছোট হাতের করে"Hi".toUpperCase() → "HI"
trim()শুরু ও শেষের Space বাদ দেয়" hi ".trim() → "hi"
split(regex)Delimiter দিয়ে ভেঙে String[] বানায়"a,b,c".split(",") → [a, b, c]
replace(old,new)সব old-কে new দিয়ে বদলায়"aa".replace('a','b') → "bb"
contains()Substring আছে কিনা Check করে"hello".contains("ell") → true
startsWith() / endsWith()শুরুতে/শেষে দেওয়া অংশ আছে কিনা"hello".startsWith("he") → true
indexOf() / lastIndexOf()প্রথম/শেষ Occurrence-এর Index, না পেলে -1"hello".indexOf('l') → 2
toCharArray()String-কে char[]-এ ভাঙে"hi".toCharArray() → ['h','i']

String Immutability — কেন Concatenation-এ নতুন String তৈরি হয়

Immutability উদাহরণ
String name = "Navin";
name = name + " Reddy";  // পুরোনো "Navin" বদলায় না — নতুন "Navin Reddy" তৈরি হয়

System.out.println(name);  // Navin Reddy
কেন গুরুত্বপূর্ণ: Loop-এর ভেতর বারবার + দিয়ে String জোড়া দিলে প্রতিবার নতুন Object তৈরি হয় — Performance খারাপ হয়। তখন Mutable StringBuilder ব্যবহার করা উচিত।

Recursion

আগে Array ও String-এর Basic Problem শেষ করে এসো। Recursion-এর অনেক Problem আসলে Array/String-এর উপরেই হয় — এবং Recursion ঠিকভাবে শিখলে পরে Tree, Backtracking ও Dynamic Programming অনেক সহজ লাগবে।

Recursion মানে — একটি Function নিজেই নিজেকে Call করে। প্রতিটা Recursive Function-এ দুটো অংশ থাকতেই হয়: একটা Base Case (যেখানে থেমে যায়) এবং একটা Recursive Case (যেখানে নিজেকে আবার Call করে, কিন্তু ছোট Input নিয়ে)।

Factorial — Base Case + Recursive Case
int factorial(int n) {
    if (n == 0) return 1;        // Base Case — এখানে থেমে যায়
    return n * factorial(n - 1); // Recursive Case — ছোট Input নিয়ে নিজেকে Call করে
}

Call Stack — Recursion Memory-তে কীভাবে কাজ করে

প্রতিটা Recursive Call-এর জন্য একটা নতুন Stack Frame তৈরি হয়, ঠিক সাধারণ Method Call-এর মতোই। যতক্ষণ Base Case না পাওয়া যায়, ততক্ষণ নতুন Frame Push হতেই থাকে — তারপর Base Case থেকে একে একে Frame Pop হয়ে Result উপরে ফেরত যায়।

CALL STACK — factorial(3)
▲ factorial(0) ← সবার শেষে Push, সবার আগে Pop (Base Case)
return=1
factorial(1)
n=1// অপেক্ষায় — 1 * factorial(0)
factorial(2)
n=2// অপেক্ষায় — 2 * factorial(1)
▼ factorial(3) ← সবার আগে Push, সবার শেষে Pop
n=3// অপেক্ষায় — 3 * factorial(2)
Unwind — Base Case থেকে ফেরত আসা
factorial(3)
= 3 * factorial(2)
= 3 * (2 * factorial(1))
= 3 * (2 * (1 * factorial(0)))
= 3 * (2 * (1 * 1))
= 6

Stack Overflow

Base Case না থাকলে বা ভুল Base Case লিখলে Function নিজেকে থামা ছাড়াই Call করতে থাকে — প্রতিটা Call-এর জন্য নতুন Frame Push হয়, একসময় Stack ভরে গিয়ে Error আসে।

Base Case ছাড়া — ভুল Recursion
int badRecursion(int n) {
    return badRecursion(n - 1);   // Base Case নেই — কখনো থামবে না
}

// Exception in thread "main" java.lang.StackOverflowError
নিয়ম: Base Case সবসময় সবার আগে লেখো, এবং নিশ্চিত করো প্রতিটা Recursive Call যেন Base Case-এর দিকে এগিয়ে যায় (যেমন n ছোট হতে হতে 0-তে পৌঁছাবে)।

Dry Run — সবচেয়ে গুরুত্বপূর্ণ Skill

Dry Run মানে কাগজে-কলমে বা মনে মনে প্রতিটা Call ধাপে ধাপে Trace করা। Recursion বোঝার এটাই আসল উপায় — Code দেখে না বুঝলে, ছোট Input নিয়ে হাতে করে দেখো।

  1. ছোট একটা Input নাও (যেমন n = 3 বা 4)
  2. প্রতিটা Call আলাদা লাইনে লিখে Stack-এর মতো সাজাও
  3. Base Case না পাওয়া পর্যন্ত Call চালিয়ে যাও
  4. Base Case থেকে Return শুরু করে, Unwind করে উপরে ওঠো
  5. প্রতি ধাপে Return Value মিলিয়ে দেখো — এখান থেকেই Pattern চোখে পড়বে

  3. factorial(4)-এর পুরো Call Stack কাগজে এঁকে Dry Run করো।

04

Linked List

Recursion না শিখে এখানে এসো না। Linked List-এর অনেক Solution (Reverse, Merge, Recursive Traversal) Recursion-এর উপর নির্ভর করে। তবে প্রথমে Iterative Solution শিখবে, তারপর Recursive।

Linked List হলো একগুচ্ছ Node-এর Chain — প্রতিটা Node-এ থাকে Data এবং পরের Node-এর ঠিকানা (next)। Array-এর মতো Continuous Memory লাগে না — প্রতিটা Node Heap-এর যেকোনো জায়গায় থাকতে পারে, শুধু next দিয়ে একটার সাথে আরেকটা জোড়া লাগানো থাকে।

Node — Linked List-এর একক
class Node {
    int data;
    Node next;

    Node(int data) {
        this.data = data;
        this.next = null;
    }
}
তিনটে Node জোড়া লাগানো
Node head = new Node(10);
head.next = new Node(20);
head.next.next = new Node(30);

// Traverse এবং Print
Node temp = head;
while (temp != null) {
    System.out.print(temp.data + " → ");
    temp = temp.next;
}
System.out.println("null");   // 10 → 20 → 30 → null
HEAP — Node গুলো ছড়িয়ে ছিটিয়ে, next দিয়ে জোড়া লাগানো
Node 1 (Head)
data=10
next=ref →
Node 2
data=20
next=ref →
Node 3 (Tail)
data=30
next=null
Head মানে প্রথম Node-এর Reference — এটা হারিয়ে গেলে পুরো Linked List হারিয়ে যায়। Tail মানে শেষ Node, যার next সবসময় null।

Singly, Doubly ও Circular Linked List

ধরনবৈশিষ্ট্য
Singly Linked Listপ্রতিটা Node শুধু পরের Node-কে চেনে (next)
Doubly Linked Listপ্রতিটা Node আগের ও পরের দুটো Node-কেই চেনে (prev + next)
Circular Linked Listশেষ Node আবার Head-কে Point করে — কোথাও null নেই

Array vs Linked List

ArrayLinked List
MemoryContinuousছড়িয়ে ছিটিয়ে (Scattered)
SizeFixedDynamic — দরকারমতো বাড়ে
Random Access (arr[i])O(1)O(n) — Head থেকে হেঁটে যেতে হয়

গুরুত্বপূর্ণ Pattern ১ — Slow & Fast Pointer

Middle Node খুঁজতে বা Loop Detect করতে সবচেয়ে ব্যবহৃত Pattern। slow এক ঘর করে এগোয়, fast দুই ঘর করে — fast শেষে পৌঁছালে slow ঠিক Middle-এ থাকে।

Middle Node খোঁজা
Node slow = head, fast = head;

while (fast != null && fast.next != null) {
    slow = slow.next;         // ১ ঘর
    fast = fast.next.next;    // ২ ঘর
}
// fast শেষে পৌঁছালে slow মাঝখানে
Floyd's Cycle Detection: একই Slow-Fast Pointer দিয়ে Loop আছে কিনা বোঝা যায় — Loop থাকলে fast কোনো একসময় ঘুরে এসে slow-কে ধরে ফেলবে (slow == fast)।

গুরুত্বপূর্ণ Pattern ২ — Reverse Linked List (Iterative)

সবচেয়ে বেশি জিজ্ঞেস করা Interview Question। তিনটে Pointer লাগে — prev, curr, next।

prev / curr / next — তিনটে Pointer
Node prev = null;
Node curr = head;

while (curr != null) {
    Node next = curr.next;   // পরের Node হারিয়ে যাওয়ার আগে ধরে রাখো
    curr.next = prev;        // Link উল্টে দাও
    prev = curr;              // prev এগিয়ে যাক
    curr = next;              // curr এগিয়ে যাক
}

head = prev;   // নতুন Head এখন শেষের Node
